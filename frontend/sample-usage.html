VocalTwist

Voice middleware integration demo — two usage patterns

Example 1 — Web Component

Drop <vocal-twist> anywhere in your HTML. Hold the mic button to record (push-to-talk) or add the ambient attribute for always-on listening.

Listening to component events

const el = document.querySelector('vocal-twist');

// Transcript received from STT
el.addEventListener('vt:transcript', (e) => {
  console.log('Transcript:', e.detail.text);
});

// TTS started playing
el.addEventListener('vt:speaking', () => {
  console.log('TTS is speaking');
});

// State changed (idle / recording / transcribing / speaking / ambient-*)
el.addEventListener('vt:statechange', (e) => {
  console.log('State:', e.detail.state);
});

// Error occurred
el.addEventListener('vt:error', (e) => {
  console.error('Error:', e.detail.error);
});

// Access the underlying VocalTwist API directly
el.vocalTwist.speak('Hello from the programmatic API!');
el.vocalTwist.setLanguage('fr');

Example 2 — Imperative API

Use VocalTwist directly when you need full control over UI and lifecycle — e.g. in a React/Vue/Angular component. 

const vt = new VocalTwist({
  transcribeUrl : '/api/transcribe',
  speakUrl      : '/api/speak',
  ambientUrl    : '/api/transcribe-ambient',
  language      : 'en',
  voice         : 'alloy',           // optional TTS voice
  apiKey        : 'sk-...',          // optional — X-API-Key header
  silenceMs     : 15000,             // ambient: flush after 15 s silence
  maxBufferMs   : 30000,             // ambient: hard flush every 30 s

  onTranscript  : (text, displayText) => console.log(text),
  onStateChange : (state) => updateMyUI(state),
  onSpeechStart : () => { /* user started speaking → interrupt TTS */ },
  onError       : (err) => console.error(err),
  onTTSStart    : () => console.log('TTS playing'),
  onTTSEnd      : () => console.log('TTS done'),
});
